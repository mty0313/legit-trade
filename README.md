# LegitTrade

Fabric 1.20.1 交易 mod。通过放置 `trade_block` 打开交易界面，用低价值物品换取高价值物品，并获得经验奖励。

## 构建

```bash
./gradlew build
```

当前版本产物：`build/libs/legittrade-1.0.1.jar`

## 使用

- 放置 `trade_block` 方块，右键打开交易界面
- 点击左侧交易项，系统会尝试从玩家背包自动填充输入槽
- 输入满足条件时，右侧会显示输出预览
- 点击输出槽执行交易
- `/tradereload` - 服务端热重载配置，并同步给在线玩家

## 交易界面

1. 左侧显示分组交易列表
2. 选择某条交易后，服务端会尝试自动填充输入槽
3. 输入槽内容变化时，输出预览会自动刷新
4. 点击输出槽后，服务端执行真实交易
5. 交易成功时：扣除输入物品，给予输出物品与经验，并播放村民成功音效

## 配置

路径：`config/legittrade.json`

当前推荐使用分组格式：

```json
[
  {
    "group": "建材",
    "trades": [
      {
        "input": "minecraft:dirt",
        "output": "minecraft:stone",
        "inputCount": 1,
        "outputCount": 64,
        "xpReward": 500
      }
    ]
  }
]
```

也兼容旧的扁平数组格式，但新配置建议使用分组格式。

字段说明：
- `group` - 分组名称
- `trades` - 该分组下的交易列表
- `input` - 输入物品 ID，必须是已注册的有效物品
- `output` - 输出物品 ID，必须是已注册的有效物品
- `inputCount` - 消耗数量，范围 `1~64`
- `outputCount` - 获得数量，范围 `1~64`
- `xpReward` - 经验奖励，必须 `>= 0`

### 配置验证与限制

加载时会自动：

- 过滤无效条目并记录警告日志
- 过滤不存在的物品 ID
- 将数量字段限制在 `1~64`
- 忽略重复交易条目
- 在配置为空或全部无效时回退到默认配置

当前限制：

- 最大分组数：`128`
- 每组最大交易数：`1024`
- 物品 ID 最大长度：`128`
- 分组名最大长度：`64`

## 技术说明

- 交易执行由服务端负责，避免客户端伪造结果
- 玩家加入服务器时会自动同步交易配置
- 执行 `/tradereload` 后会重新同步给所有在线玩家
- 客户端断开连接时会清空本地缓存的交易配置
- `trade_block` 已包含配方、模型、语言文件和掉落表
